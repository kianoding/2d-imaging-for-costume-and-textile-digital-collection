# 2D Imaging Workflow SOP
**Costume & Textile Study Collection**

---

## Overview

This handbook defines a practical workflow for digitizing garments and textile items in the study collection. It is designed for use in a small-team environment without a dedicated imaging department, where digitization is carried out as part of ongoing archival work.

The workflow addresses the need for clear, scalable procedures that produce digital files that are consistent, usable, and sustainable over time [1]. It documents an end-to-end process: from image capture through processing and file and metadata preparation. Hence, digitization work can be performed reliably by different users.

Digitization is understood as the creation of a digital representation of a physical object, while digital collections are organized groups of these files intended to support access and use. Digital preservation extends this process by maintaining digital assets to ensure their long-term usability [1]. This workflow situates imaging within that broader system.

---

## The Workflow (At a Glance)

All items follow the same core process:

```
[Backlog]
    ↓
[Planning & Grouping]
    ↓
[Preparation]
    ↓
[Imaging]
    ↓
[Post-Processing]
    ↓
[Output & Storage]
```

Each stage builds on the previous one. Incomplete or inconsistent work at any stage will affect the final digital object.

## Roles and Responsibilities

This workflow is designed for a small team with overlapping responsibilities.

- **Photographer**: image capture, processing, and technical quality control
- **Preparators / Catalogers**: object handling, mounting, measurements, and metadata

In practice, the Archive Assistant may perform multiple roles, including preparation, cataloging, quality control, and coordination. The workflow is structured to support this flexibility while maintaining consistency across all stages.

---

## Scope

This handbook covers:

- Planning and execution of studio imaging
- Image processing and quality control
- Preparation of preservation and access files
- Identification of essential metadata
- Workflow documentation in a modular format [1]

It does not include accessioning workflows or conservation treatment, and assumes items are already present in the collection inventory [1].

## How to Use This Handbook

This is a procedural guide.

- Complete each stage before moving forward
- Use individual pages for step-by-step instructions

The goal is to ensure that all digitized items move through a consistent and repeatable system.

## Documentation Model
 
This handbook is structured as a modular, bite-sized guide — one page per workflow stage, each self-contained and independently maintainable. This approach is inspired by the **Society of American Archivists' *Archival Accessioning Best Practices*** (ABP v.1.0.2), which uses a similar modular GitBook structure to document archival workflows for small teams [6].
 
This handbook is hosted on **GitHub** rather than GitBook, using Markdown files that render natively — providing the same modular, navigable structure with the added benefit of version control and no platform dependency.
